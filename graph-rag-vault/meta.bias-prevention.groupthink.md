# meta.bias-prevention.groupthink

> 집단사고(Groupthink)는 Irving Janis(1972)가 정의한 현상으로, 집단의 조화를 유지하려는 욕구가 비판적 사고를 억압하여 결함 있는 의사결정을 내리게 만든다. 피그만 침공이 대표 사례이다.

# 집단사고 방지 가이드

## 정의
팀의 합의와 조화를 지키려는 욕구가 비판적 사고를 억압하는 현상.
예: 리더가 방향을 정하면 아무도 반대 의견을 내지 않음.

## 발생 징후
- 만장일치 환상: "모두 동의하죠?"에 침묵 = 동의
- 자기 검열: 다른 의견을 가져도 말하지 않음
- 이탈자 압력: 반대 의견에 "팀플레이 안 하네" 반응
- 외부 의견 무시: "그 사람들은 우리 상황을 몰라"

## 방지 전략
1. **악마의 대변인**: 매 회의마다 반대 역할 지정 (순환)
2. **익명 투표**: 의견 표출 전 개별 의견 먼저 수집
3. **외부 전문가 초대**: 팀 외부 시각 정기적 도입
4. **리더 최후 발언**: 리더가 먼저 의견을 말하지 않기

## 체크리스트
- [ ] 팀에서 반대 의견이 자유롭게 나오는가?
- [ ] 리더가 먼저 의견을 밝히지 않았는가?
- [ ] 외부 관점을 검토했는가?
- [ ] 만장일치가 진짜 합의인지 확인했는가?

## DO
- 회의 시작 시 각자 의견을 먼저 기록 후 공유
- "이 결정의 가장 큰 위험은?" 질문 루틴화
- 심리적 안전성 정기 측정 및 개선

## DON'T
- 리더의 의견에 자동 동의하지 않기
- 빠른 합의를 좋은 합의로 착각하지 않기
- 반대 의견을 팀워크 저해로 프레이밍하지 않기
