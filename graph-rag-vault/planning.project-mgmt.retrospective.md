# planning.project-mgmt.retrospective

> 회고(Retrospective)는 스프린트/프로젝트 종료 후 팀이 프로세스를 돌아보고 개선점을 도출하는 애자일 의식이다. Esther Derby의 5단계 모델(준비→데이터수집→인사이트도출→액션결정→마무리)이 대표적이다.

# 회고(Retrospective) 가이드

## 핵심 원칙
- 사람이 아닌 프로세스에 초점
- 심리적 안전성이 전제 조건
- 액션 아이템 없는 회고는 무의미
- 다양한 포맷으로 참여도 유지

## 5단계 프로세스
1. **준비** (5분): 회고 목적, 그라운드 룰 공유
2. **데이터 수집** (15분): 스프린트 사실 기반 수집
3. **인사이트 도출** (15분): 패턴 발견, 근본 원인 탐색
4. **액션 결정** (10분): 다음 스프린트 실행 항목 합의
5. **마무리** (5분): 회고에 대한 회고

## 회고 포맷
| 포맷 | 구조 | 적합 |
|------|------|------|
| Start-Stop-Continue | 시작/중단/지속할 것 | 범용 |
| 4Ls | Liked/Learned/Lacked/Longed | 긍정적 팀 |
| Mad-Sad-Glad | 감정 기반 분류 | 감정 표현 유도 |
| 범선(Sailboat) | 바람(추진)/닻(저해)/암초(위험) | 시각적 |
| 타임라인 | 시간순 사건 나열 | 긴 프로젝트 |

## 액션 아이템 기준
- 구체적: "커뮤니케이션 개선" X → "PR 리뷰 24시간 내 완료 규칙" O
- 측정 가능: 다음 회고에서 달성 여부 확인 가능
- 담당자 배정: 주인 없는 액션은 실행되지 않음
- 2-3개로 제한: 과도한 액션은 실행률 저하

## DO
- 회고 전 익명 설문으로 사전 데이터 수집
- 이전 회고 액션 아이템 실행 여부 점검
- 포맷을 주기적으로 변경

## DON'T
- 비난이나 개인 공격 허용하지 않기
- 액션 아이템 없이 회고 종료하지 않기
- 매번 같은 포맷만 사용하지 않기
