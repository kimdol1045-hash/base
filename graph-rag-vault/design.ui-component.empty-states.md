# design.ui-component.empty-states

> 빈 상태(Empty States)는 콘텐츠가 없는 화면의 디자인으로, 사용자 경험의 첫인상과 온보딩에 결정적 영향을 미친다. 단순히 '데이터 없음'이 아닌 가치 제안과 행동 유도의 기회로 활용해야 한다.

# 빈 상태 디자인 가이드

## 핵심 원칙
- 빈 상태는 온보딩의 기회
- 사용자에게 다음 행동을 명확히 안내
- 감정적 연결을 위한 일러스트 활용
- 상황별 다른 빈 상태 메시지

## 빈 상태 유형
| 유형 | 상황 | 접근 방식 |
|------|------|----------|
| 첫 사용 | 아직 데이터 없음 | 가치 제안 + CTA |
| 검색 결과 없음 | 조건 불일치 | 대안 제안 + 필터 초기화 |
| 에러 | 로드 실패 | 원인 설명 + 재시도 |
| 빈 목록 | 항목 삭제됨 | 새 항목 생성 유도 |
| 완료 | 모든 작업 완료 | 축하 + 다음 단계 |

## 구성 요소
1. **일러스트/아이콘**: 시각적 맥락 제공
2. **제목**: 현재 상태 설명 (1줄)
3. **설명**: 다음 행동 안내 (1-2줄)
4. **CTA 버튼**: 명확한 행동 유도

## DO
- 첫 사용 빈 상태에서 제품의 핵심 가치 전달
- 검색 결과 없음 시 대안 키워드 제안
- 브랜드 톤에 맞는 친근한 문구 사용

## DON'T
- 빈 화면을 그대로 두지 않기
- "데이터가 없습니다"만 표시하지 않기
- 기술적 에러 코드를 사용자에게 노출하지 않기
- 너무 많은 텍스트로 빈 상태를 채우지 않기
