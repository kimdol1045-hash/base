# planning.business.design-sprint

> #168 Design Sprint (Jake Knapp, Google Ventures, 2016)

디자인 스프린트 (5일 만에 아이디어를 검증한다):

### 5일 프로세스
| 요일 | 활동 | 산출물 |
|------|------|--------|
| 월 | Map: 문제 정의 + 장기 목표 | 문제 맵, 스프린트 질문 |
| 화 | Sketch: 개별 솔루션 아이디어 | 솔루션 스케치 (Crazy 8s) |
| 수 | Decide: 최적 솔루션 선택 | 스토리보드 |
| 목 | Prototype: 테스트용 프로토타입 | 클릭 가능한 프로토타입 |
| 금 | Test: 5명 사용자 테스트 | 학습 결과 + 의사결정 |

### 핵심 도구
- How Might We: "어떻게 하면 ~할 수 있을까?" 질문 포맷
- Crazy 8s: 8분간 8개 솔루션 스케치
- Dot Voting: 민주적 의사결정
- 5명 테스트: Nielsen의 5명이면 85% 이슈 발견 법칙

### 적용 시점
- ✅: 새 제품/기능 시작, 큰 의사결정 앞둠, 팀 합의 필요
- ❌: 이미 방향이 명확, 기술적 문제만 있음, 데이터로 답 나옴

### 간소화 버전 (3일 스프린트)
- Day 1: Map + Sketch + Decide
- Day 2: Prototype
- Day 3: Test
