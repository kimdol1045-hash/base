# qa.ux-audit.cognitive-walkthrough

> 인지적 워크스루 — 사용자 관점 태스크 수행 평가

# 인지적 워크스루 가이드

## 핵심 원칙
- 대표 태스크 정의 → 단계별 사용자 관점 평가
- 각 단계: "사용자가 올바른 행동을 찾을 수 있는가?"
- 학습 가능성과 직관성 중심 평가

## DO
- 핵심 사용자 여정 3-5개 선정
- 각 단계별 4가지 질문 체크 (목표, 행동, 피드백, 진행)
- 신규 사용자 관점에서 평가

## DON'T
- 전문 사용자 기준으로만 평가하지 않기
- 미적 요소만 평가하고 태스크 완수율 무시하지 않기
- 발견된 문제를 심각도 분류 없이 나열하지 않기
