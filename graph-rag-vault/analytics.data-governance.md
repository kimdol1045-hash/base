# analytics.data-governance

> 데이터 거버넌스는 데이터의 품질, 보안, 접근성, 규정 준수를 관리하는 체계이다. DAMA-DMBOK 프레임워크를 기반으로 데이터 카탈로그, 품질 규칙, 접근 정책, 계보 관리를 포함한다.

# 데이터 거버넌스 가이드

## 핵심 원칙
- 데이터 품질: 정확성, 완전성, 일관성, 적시성
- 데이터 보안: 접근 제어, 암호화, 감사 로그
- 데이터 계보: 원천부터 소비까지 추적 가능
- 규정 준수: GDPR, PIPA 등 법적 요건 충족

## 거버넌스 프레임워크
1. **데이터 카탈로그**: 모든 데이터셋의 메타데이터 관리
2. **데이터 소유자**: 각 데이터셋의 책임자 지정
3. **품질 규칙**: 자동화된 데이터 품질 검증
4. **접근 정책**: 역할 기반 접근 제어(RBAC)
5. **계보 관리**: ETL 파이프라인 추적
6. **보존 정책**: 데이터 생명주기 관리

## 데이터 품질 차원
| 차원 | 정의 | 측정 방법 |
|------|------|----------|
| 정확성 | 실제 값과 일치 | 교차 검증 |
| 완전성 | NULL/누락 비율 | NULL 비율 모니터링 |
| 일관성 | 시스템 간 동일 | 소스 간 비교 |
| 적시성 | 최신 상태 유지 | 갱신 지연 시간 |

## DO
- 데이터 카탈로그를 모든 분석가가 접근 가능하게 유지
- PII 데이터 마스킹/익명화 자동화
- 데이터 품질 대시보드 운영

## DON'T
- 데이터 정의 없이 분석 시작하지 않기
- PII를 개발/분석 환경에 그대로 복사하지 않기
- 데이터 품질 이슈를 방치하지 않기
