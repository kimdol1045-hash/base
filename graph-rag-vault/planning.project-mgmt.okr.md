# planning.project-mgmt.okr

> #167 OKR (Andy Grove → John Doerr, Measure What Matters, 2018)

OKR (목표와 핵심 결과로 팀을 정렬한다):

### 구조
- Objective: 정성적 목표 (영감을 주는 방향)
- Key Result: 정량적 측정 지표 (2~5개)

### 좋은 OKR 예시
```
Objective: 사용자 온보딩 경험을 혁신한다

KR1: 가입→첫 가치 도달 시간을 5분 → 2분으로 단축
KR2: Day 7 리텐션을 25% → 40%로 개선
KR3: 온보딩 완료율을 60% → 85%로 향상
```

### 나쁜 OKR 안티패턴
| 안티패턴 | 예시 | 문제 |
|---------|------|------|
| 산출물 = 결과 | "온보딩 화면 3개 제작" | 산출물이지 결과가 아님 |
| 측정 불가 | "UX 개선" | 어떻게 측정? |
| 너무 쉬움 | "리텐션 1% 개선" | 도전적이지 않음 |
| KR 10개 | 집중력 분산 | 2~5개가 적정 |

### 주기 및 리뷰
- 분기별 설정 (연간 OKR + 분기 OKR)
- 주간 체크인 (진행률 0~1.0)
- 분기말 채점: 0.6~0.7 = 건강한 달성 (1.0은 목표가 너무 쉬웠다는 의미)

### vs KPI
| OKR | KPI |
|-----|-----|
| 도전적 목표 + 변화 | 현재 성과 추적 |
| 분기별 갱신 | 상시 모니터링 |
| 70% 달성이 성공 | 100% 달성이 기준 |
