# planning.user-persona.empathy-map

> **Region**: 🧠 [[PREFRONTAL]]  
> **Domain**: `planning`  
> **Type**: `pattern`

## Content

Empathy Map (사용자의 머릿속으로 들어가 진짜 니즈를 발견한다):

### 왜 Empathy Map인가?
페르소나가 "누구인가"를 정의한다면, Empathy Map은 "무엇을 경험하는가"를
깊이 탐구한다. 사용자가 말하는 것과 실제로 생각하는 것 사이의 갭에서
가장 강력한 제품 인사이트가 나온다. 표면적 요구가 아닌 잠재적 니즈를
발견하기 위한 도구이다.

### Empathy Map 4사분면

```
┌──────────────────────┬──────────────────────┐
│ Says (말하는 것)      │ Thinks (생각하는 것)  │
│                      │                      │
│ 인터뷰, 리뷰에서     │ 말하지 않지만         │
│ 직접 표현한 말        │ 속으로 생각하는 것     │
│                      │                      │
├──────────────────────┼──────────────────────┤
│ Does (행동하는 것)    │ Feels (느끼는 것)     │
│                      │                      │
│ 관찰 가능한           │ 감정 상태,            │
│ 실제 행동 패턴        │ 두려움, 기대감        │
│                      │                      │
└──────────────────────┴──────────────────────┘
```

### 각 사분면 작성 가이드

**Says (말하는 것)** — 3-5개
데이터 소스: 인터뷰, 앱스토어 리뷰, 고객 문의, 소셜 미디어
- 사용자가 직접 표현한 말을 인용구 형태로 기록
- DO: "매번 같은 작업을 반복하는 게 너무 귀찮아요"
- DO: "다른 앱을 써봤는데 다 비슷비슷해요"
- DON'T: "사용자는 효율적인 워크플로우를 원한다" (연구자의 해석, 사용자의 말이 아님)

**Thinks (생각하는 것)** — 3-5개
데이터 소스: 행동과 말의 차이에서 추론, 맥락 분석
- 사용자가 겉으로 표현하지 않지만 속으로 생각하는 것
- Says와 Thinks 사이의 "갭"이 핵심 인사이트
- DO: "이 앱이 내 데이터를 어떻게 사용하는지 좀 불안하다..."
- DO: "무료 버전으로 충분한데 유료 전환을 계속 유도하네..."
- 갭 예시:
  Says: "이 앱 괜찮아요, 잘 쓰고 있어요"
  Thinks: "다른 앱이 나오면 바로 옮길 텐데..."
  → 갭 인사이트: 만족이 아닌 관성으로 사용 중. 전환 비용이 낮다.

**Does (행동하는 것)** — 3-5개
데이터 소스: 사용 로그, 행동 관찰, 히트맵
- 실제 관찰 가능한 행동 패턴
- 말과 행동의 차이에 주목
- DO: "매일 아침 출근길에 앱을 열지만 3분 이내에 닫는다"
- DO: "검색 기능 대신 최근 항목 목록만 사용한다"
- DON'T: "앱을 효율적으로 사용한다" (관찰이 아닌 평가)

**Feels (느끼는 것)** — 3-5개
데이터 소스: 표정, 어조, 감탄사, 리뷰의 감정 어휘
- 감정 상태와 그 원인
- 긍정적 감정과 부정적 감정 모두 포함
- DO: "월말 정산 때 불안함 — 혹시 빠뜨린 항목이 있을까 걱정"
- DO: "새 기능 발견 시 작은 기쁨 — '오 이런 것도 되네!'"
- DON'T: "좋은 경험을 느낀다" (구체적 감정과 원인 누락)

### 인사이트 도출법

**Step 1: 갭 분석**
4사분면을 교차 비교하여 모순과 갭을 찾는다:
- Says vs Does: "간편하다고 말하지만 실제로는 5단계를 거친다"
- Says vs Thinks: "만족한다고 하지만 불안감을 느끼고 있다"
- Thinks vs Feels: "합리적으로 괜찮다고 생각하지만 감정적으로 불만"
- Does vs Feels: "반복 사용하지만 매번 짜증을 느낀다"

**Step 2: 니즈 도출**
갭에서 Unmet Need를 추출한다:
```
갭: Says "잘 쓰고 있다" vs Does "핵심 기능을 30%만 사용"
인사이트: 기능 발견성(Discoverability)이 낮다
니즈: 사용자가 자연스럽게 고급 기능을 발견하는 온보딩 필요
```

**Step 3: 기회 우선순위**
| 인사이트 | 영향받는 사용자 비율 | Pain 강도 | 해결 난이도 | 우선순위 |
|----------|---------------------|-----------|-------------|----------|

### Empathy Map 작성 템플릿
```
페르소나: [이름]
맥락: [어떤 상황에서의 경험인지]

┌─ Says ──────────────────┬─ Thinks ─────────────────┐
│ 1. "                  " │ 1. (                   )  │
│ 2. "                  " │ 2. (                   )  │
│ 3. "                  " │ 3. (                   )  │
├─ Does ──────────────────┼─ Feels ──────────────────┤
│ 1.                      │ 1.          (감정: )      │
│ 2.                      │ 2.          (감정: )      │
│ 3.                      │ 3.          (감정: )      │
└─────────────────────────┴──────────────────────────┘

핵심 갭:
- [Says vs Does/Thinks 갭 서술]

인사이트:
- [갭에서 도출된 Unmet Need]

제품 기회:
- [인사이트를 해결하는 기능/개선 방향]
```

### 주의사항
- Empathy Map은 특정 "맥락"에 대해 작성한다 (전체 라이프가 아닌 특정 상황)
- 실제 데이터 없이 작성 시 "[가설 기반]" 태그 부착
- Says 사분면에 연구자의 해석을 넣지 말 것 → 사용자의 실제 표현만
- Says와 Thinks가 완전히 동일하면 분석 깊이가 부족한 것
