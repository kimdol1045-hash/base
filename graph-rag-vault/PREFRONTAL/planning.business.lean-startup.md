### Build-Measure-Learn 루프
1. **Build**: 최소 기능 제품(MVP) 구축. 가설 검증에 필요한 것만.
2. **Measure**: 핵심 지표 수집. 허영 지표(페이지뷰) ❌ → 실행 지표(전환율) ✅
3. **Learn**: 데이터 기반 의사결정. Pivot(방향 전환) 또는 Persevere(유지).

### 린 캔버스 템플릿
| 항목 | 내용 |
|------|------|
| 1. 문제 | 상위 3가지 문제 |
| 2. 고객 세그먼트 | 초기 타겟 (얼리어답터) |
| 3. 고유 가치 제안 | 왜 이것을 써야 하는지 한 문장 |
| 4. 솔루션 | 각 문제에 대한 해결책 |
| 5. 채널 | 고객에게 도달하는 경로 |
| 6. 수익 모델 | 어떻게 돈을 버는지 |
| 7. 비용 구조 | 핵심 비용 항목 |
| 8. 핵심 지표 | 성공을 측정하는 1-2개 지표 |
| 9. 경쟁 우위 | 쉽게 모방할 수 없는 것 |

### Pivot 판단 기준
- 3개월 내 핵심 지표 개선 없으면 Pivot 고려
- Pivot ≠ 실패. 학습 기반 방향 전환.
- Pivot 유형: 고객 세그먼트, 채널, 수익 모델, 기능, 플랫폼
