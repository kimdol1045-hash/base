## Content

유저 페르소나 자기 검증 체크리스트 (페르소나 작성 후 반드시 실행):

### 왜 페르소나 검증이 필요한가?
잘못된 페르소나는 팀 전체를 잘못된 사용자 상으로 이끈다.
"모든 사람을 위한 제품"은 "아무도 위하지 않는 제품"이 되기 쉽다.
페르소나가 충분히 구체적이고 차별화되었는지 검증이 필수적이다.

### 구체성 검증
- [ ] 이름, 나이, 직업이 구체적으로 설정되어 있는가?
  → "30대 직장인" (X) vs "김서연, 33세, 스타트업 마케팅 매니저" (O)
- [ ] 하루 일과가 시간대별로 서술되어 있는가?
- [ ] 사용하는 앱/서비스가 3-5개 구체적으로 나열되어 있는가?
- [ ] 대표 인용구가 실제 사람이 할 법한 말인가?
  → "효율적인 솔루션이 필요합니다" (X, 마케팅 카피)
  → "또 깜빡하고 메모 안 했네... 나중에 기억이 안 날 텐데" (O, 실제 말투)

### 니즈 검증
- [ ] 핵심 니즈가 "제품 기능"이 아닌 "문제/고통" 중심으로 정의되었는가?
  → "검색 기능이 필요하다" (X, 기능)
  → "원하는 정보를 3분 안에 찾고 싶다" (O, 니즈)
- [ ] Pain Point가 3가지 이상 구체적으로 서술되어 있는가?
- [ ] JTBD 형식으로 니즈가 표현되어 있는가?
- [ ] 현재 대안의 불편함이 구체적 상황과 함께 서술되어 있는가?

### 차별화 검증
- [ ] Primary와 Secondary 페르소나가 서로 다른 세그먼트인가?
- [ ] 두 페르소나 간 차이점이 최소 2가지 이상인가?
  (행동 패턴, 기술 친숙도, 핵심 니즈 중 2가지)
- [ ] "나이만 다른 같은 사람"이 아닌가?
- [ ] 각 페르소나가 제품을 사용하는 맥락이 다른가?

### 현실성 검증
- [ ] 페르소나가 "이상적 고객"이 아닌 "현실적 고객"인가?
  → 모든 기능을 사용하고 매일 접속하는 완벽한 고객 (X)
  → 특정 상황에서만 사용하고 가끔 잊어버리는 현실적 고객 (O)
- [ ] 전환 저항(Habit, Anxiety)이 포함되어 있는가?
- [ ] 기술 친숙도가 상/중/하로 설정되고 행동과 일관성이 있는가?
  → 기술 친숙도 "하"인데 Slack, Notion, Figma를 사용하는 것은 모순
- [ ] 소비 성향이 제품 가격 전략과 맞는가?

### 행동 패턴 검증
- [ ] 행동 패턴이 시간대별로 구체적인가?
- [ ] 제품 사용 맥락(언제, 어디서, 왜)이 명확한가?
- [ ] 현재 어떤 도구/방법을 사용하는지 서술되어 있는가?
- [ ] 주중과 주말 행동 차이가 고려되어 있는가? (해당 시)

### Empathy Map 검증 (해당 시)
- [ ] Says/Thinks/Does/Feels 4사분면이 모두 채워져 있는가?
- [ ] Says와 Thinks 사이에 의미 있는 갭이 있는가?
  (말하는 것과 생각하는 것이 항상 같으면 인사이트가 부족하다)
- [ ] 인사이트가 4사분면의 패턴에서 도출되었는가?

### 검증 실패 시 조치
- 구체성 미통과: 인구통계, 행동 패턴, 인용구를 더 구체화
- 차별화 미통과: 세그먼트 기준을 재설정하고 페르소나 재작성
- 현실성 미통과: 전환 저항과 현실적 제약 추가
- 행동 패턴 미통과: 시간대별 구체적 행동 추가
