# meta.bias-prevention.anchoring-bias

> 앵커링 편향은 처음 접한 정보(앵커)에 과도하게 의존하여 이후 판단이 왜곡되는 인지 편향이다. Tversky & Kahneman(1974)이 발견했으며, 가격 협상, 견적, 일정 추정에서 빈번히 발생한다.

# 앵커링 편향 방지 가이드

## 정의
처음 접한 숫자나 정보에 끌려 이후 판단이 왜곡되는 현상.
예: "이 프로젝트 3주면 되죠?"라는 질문 자체가 앵커가 됨.

## 발생 시나리오
- 일정 추정: 누군가의 첫 제안이 기준점이 됨
- 가격 협상: 첫 제시 가격이 최종 가격에 영향
- 성과 평가: 첫인상이 전체 평가를 좌우
- 데이터 분석: 처음 본 수치 중심으로 해석

## 방지 전략
1. **독립 추정 우선**: 외부 수치를 보기 전에 자체 추정
2. **다중 앵커**: 하나가 아닌 여러 기준점에서 시작
3. **역방향 사고**: "이 추정이 2배 틀렸다면?" 질문
4. **기저율 참조**: 과거 유사 사례 데이터 확인

## 체크리스트
- [ ] 첫 번째 들은 숫자에 영향받고 있지 않은가?
- [ ] 독립적으로 추정한 값과 비교했는가?
- [ ] 기저율(Base Rate) 데이터를 확인했는가?
- [ ] 범위 추정(최선/최악/가능)을 시도했는가?

## DO
- 추정 시 "최선/가능/최악" 3가지 시나리오 제시
- 과거 데이터 기반 비교 수치 함께 제공

## DON'T
- 첫 번째 제시된 수치를 기준으로 조정만 하지 않기
- 앵커가 있는 상태에서 팀 추정하지 않기
