# qa.ux-audit.content-audit

> UX 콘텐츠 감사 — 사용자 대면 텍스트 품질 검증

# UX 콘텐츠 감사 가이드

## 핵심 원칙
- 모든 사용자 대면 텍스트의 명확성, 일관성, 톤 검증
- 에러 메시지, 빈 상태, 로딩 텍스트 포함
- 접근성과 다국어 지원 고려

## DO
- 에러 메시지: 원인 + 해결 방법 포함 여부 확인
- CTA 버튼 텍스트의 동작 명확성 검증
- 전문 용어의 사용자 친화적 대체어 확인

## DON'T
- 개발자 용어를 사용자 메시지에 노출하지 않기
- placeholder 텍스트 ("Lorem ipsum") 방치하지 않기
- 톤앤보이스 가이드 없이 임의 작성하지 않기
