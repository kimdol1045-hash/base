## Content

와이어프레임 자기 검증 체크리스트 — 모든 항목을 통과해야 출력한다.

### A. 구조 완성도
- [ ] 모바일 (320px) 와이어프레임이 포함되어 있는가?
- [ ] 데스크톱 (1280px) 와이어프레임이 포함되어 있는가?
- [ ] 페이지 목적이 한 문장으로 정의되어 있는가?
- [ ] 사용자 시나리오가 명시되어 있는가?

### B. 정보 계층 (Information Hierarchy)
- [ ] H1이 페이지당 1개만 존재하는가?
- [ ] 제목 계층이 H1 > H2 > H3 순서를 건너뛰지 않는가?
- [ ] Above the Fold (첫 600px)에 핵심 가치 제안이 있는가?
- [ ] 3초 내에 "이 페이지가 뭔지" 파악 가능한가?
- [ ] 정보 우선순위: 가장 중요한 것이 가장 크고 상단에 있는가?

### C. CTA 전략
- [ ] 페이지에 명확한 주요 CTA가 1개 이상 있는가?
- [ ] CTA 문구가 구체적 행동을 유도하는가? ("자세히 보기" X, "무료로 시작하기" O)
- [ ] CTA가 시각적으로 가장 눈에 띄는 위치에 있는가?
- [ ] Above the Fold에 CTA가 최소 1개 있는가?
- [ ] 긴 페이지의 경우 스크롤 중간/하단에 CTA가 반복 배치되어 있는가?

### D. 시선 흐름 (Eye Flow)
- [ ] 랜딩 페이지: Z패턴 (좌상단 로고 → 우상단 CTA → 좌하단 콘텐츠 → 우하단 CTA) 적용?
- [ ] 콘텐츠 페이지: F패턴 (좌측 헤딩 스캔 → 관심 섹션 우측 읽기) 적용?
- [ ] 시선 흐름이 끊기는 구간이 없는가? (갑작스러운 레이아웃 변화)
- [ ] 네비게이션 → 히어로 → 콘텐츠 → CTA 자연스러운 흐름인가?

### E. 반응형 전환
- [ ] 모바일에서 숨겨지는 핵심 콘텐츠가 없는가?
- [ ] 모바일: 단일 컬럼 스택 레이아웃인가?
- [ ] 데스크톱: 적절한 그리드/플렉스 레이아웃인가?
- [ ] 네비게이션이 모바일에서 적절히 축소되는가? (햄버거 메뉴 등)

### F. 인지 부하 (Cognitive Load)
- [ ] 한 화면의 선택지가 5개 이하인가? (Hick's Law)
- [ ] 빈 공간(여백)이 충분한가? (요소 간 최소 32px)
- [ ] 유사한 요소가 일관된 패턴으로 반복되는가?
- [ ] 불필요한 장식 요소가 없는가? (구조와 기능에 집중)
