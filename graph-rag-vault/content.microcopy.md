# content.microcopy

> 마이크로카피는 버튼, 레이블, 툴팁, 에러 메시지 등 인터페이스의 작은 텍스트를 말한다. 짧지만 사용자 경험에 결정적 영향을 미치며, Joshua Porter의 연구에서 한 줄 마이크로카피 추가로 전환율이 160% 상승한 사례가 있다.

# 마이크로카피 가이드

## 핵심 원칙
- 짧을수록 좋다 (5단어 이내 권장)
- 사용자의 불안을 해소하는 데 집중
- 행동의 결과를 미리 알려준다
- 브랜드 톤을 유지하되 명확성 우선

## 유형별 가이드
### 버튼 텍스트
- 동사로 시작: "저장하기", "시작하기", "구독하기"
- 결과를 포함: "무료로 시작하기", "PDF 다운로드"

### 에러 메시지
- 구조: 무엇이 잘못됐는지 + 어떻게 해결하는지
- 예: "비밀번호는 8자 이상이어야 합니다. 특수문자를 포함해 주세요."

### 빈 상태 (Empty State)
- 현재 상태 설명 + 다음 행동 유도
- 예: "아직 프로젝트가 없습니다. 첫 프로젝트를 만들어 보세요."

### 폼 안내 텍스트
- 플레이스홀더가 아닌 도움말 텍스트로 가이드
- 예: "영문, 숫자 포함 8자 이상"

## DO
- 사용자가 망설이는 지점에 안심 문구 배치
- A/B 테스트로 마이크로카피 효과 검증
- 일관된 동사 시제와 어미 사용

## DON'T
- 플레이스홀더를 레이블 대신 사용하지 않기
- "여기를 클릭하세요" 같은 불명확한 텍스트 사용하지 않기
- 사용자를 비난하는 어투 사용하지 않기 ("잘못 입력했습니다")
