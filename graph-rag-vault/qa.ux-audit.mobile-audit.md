# qa.ux-audit.mobile-audit

> 모바일 감사는 모바일 기기에서의 사용자 경험을 체계적으로 평가하는 프로세스이다. 터치 인터랙션, 뷰포트 대응, 성능, 접근성을 포함하며, 모바일 트래픽이 전체의 60% 이상인 현실을 반영한다.

# 모바일 감사 가이드

## 핵심 원칙
- 모바일 퍼스트: 모바일을 기본, 데스크톱을 확장으로
- 터치 친화적: 44x44px 이상 터치 타겟
- 성능 최우선: 모바일 네트워크/기기 제약 고려
- 실제 기기 테스트 필수 (에뮬레이터만 불충분)

## 감사 체크리스트
### 레이아웃
- [ ] 뷰포트 메타 태그 설정
- [ ] 가로 스크롤 없음 (모든 뷰포트)
- [ ] 텍스트 가독성 (16px 이상, 줄 간격 1.5)
- [ ] 적절한 여백과 패딩

### 터치 인터랙션
- [ ] 터치 타겟 최소 44x44px
- [ ] 인접 터치 타겟 간 8px 이상 간격
- [ ] 스와이프, 핀치줌 등 제스처 지원
- [ ] 호버 의존 UI 없음

### 성능
- [ ] 3G 네트워크에서 3초 내 로딩
- [ ] 이미지 반응형 (srcset)
- [ ] 지연 로딩 적용
- [ ] 오프라인 기본 지원 (PWA 고려)

### 입력
- [ ] 적절한 input type (email, tel, number)
- [ ] 자동완성(autocomplete) 지원
- [ ] 키보드로 폼 제출 가능
- [ ] 소프트 키보드에 의한 레이아웃 깨짐 없음

## 테스트 기기 매트릭스
| OS | 기기 | 화면 크기 |
|----|------|----------|
| iOS | iPhone SE, 15 | 375px, 393px |
| Android | Pixel, Galaxy | 360px, 412px |
| 태블릿 | iPad | 768px |

## DO
- Chrome DevTools + 실제 기기 테스트 병행
- 느린 네트워크 시뮬레이션으로 테스트
- 세로/가로 방향 전환 테스트

## DON'T
- 데스크톱만 테스트하고 모바일 무시하지 않기
- 에뮬레이터만으로 모바일 테스트 완료하지 않기
- 터치 이벤트를 마우스 이벤트로 대체하지 않기
