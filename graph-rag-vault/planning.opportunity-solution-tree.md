# planning.opportunity-solution-tree

> OST(Opportunity Solution Tree)는 Teresa Torres가 제안한 시각적 프레임워크로, 원하는 결과(Outcome)에서 출발하여 기회(Opportunity)를 발굴하고 솔루션을 도출하는 구조이다.

# Opportunity Solution Tree 가이드

## 핵심 원칙
- 결과(Outcome)에서 시작하여 역순으로 탐색
- 기회(Opportunity)는 고객의 니즈/페인/욕구
- 하나의 기회에 여러 솔루션을 탐색
- 실험으로 솔루션 검증 후 구현

## OST 구조
```
비즈니스 목표 (Desired Outcome)
├── 기회 1 (Customer Need)
│   ├── 솔루션 A
│   │   └── 실험 A-1
│   └── 솔루션 B
│       └── 실험 B-1
├── 기회 2
│   ├── 솔루션 C
│   └── 솔루션 D
└── 기회 3
    └── 솔루션 E
```

## 작성 프로세스
1. **Outcome 정의**: 팀의 분기 목표 (OKR의 KR)
2. **기회 발굴**: 고객 인터뷰에서 니즈/페인 추출
3. **기회 구조화**: 상위/하위 기회로 계층화
4. **기회 선택**: 영향도 x 빈도로 우선순위
5. **솔루션 발상**: 선택된 기회에 3개 이상 솔루션
6. **실험 설계**: 가장 빠르게 검증할 방법
7. **반복**: 실험 결과로 트리 업데이트

## DO
- 주간 디스커버리 미팅에서 OST 업데이트
- 기회를 고객의 언어로 표현
- 솔루션 편향 방지: 기회부터 충분히 탐색

## DON'T
- 솔루션에서 시작하여 기회를 역추적하지 않기
- 기회 탐색 없이 바로 솔루션 구현하지 않기
- OST를 한 번 만들고 방치하지 않기
