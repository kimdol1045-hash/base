# design.design-system.documentation

> 디자인 시스템 문서화는 컴포넌트의 사용법, 가이드라인, 원칙을 체계적으로 기록하여 팀 전체의 일관된 사용을 보장한다. 좋은 문서가 없는 디자인 시스템은 채택률이 크게 떨어진다.

# 디자인 시스템 문서화 가이드

## 핵심 원칙
- 디자이너와 개발자 모두를 위한 문서
- 실행 가능한 코드 예시 필수
- 언제 사용하고 언제 사용하지 않는지 명시
- 문서 자체가 디자인 시스템의 라이브 데모

## 문서 구조 (컴포넌트별)
1. **개요**: 컴포넌트의 목적과 역할
2. **라이브 데모**: 인터랙티브 플레이그라운드
3. **Props/API**: 속성 테이블 (이름, 타입, 기본값, 설명)
4. **변형(Variants)**: 모든 시각적 변형 갤러리
5. **사용 가이드**: 올바른 사용법과 잘못된 사용법
6. **접근성**: 키보드, 스크린 리더 지원 명세
7. **관련 컴포넌트**: 유사하거나 함께 사용하는 컴포넌트
8. **변경 이력**: 버전별 변경 내용

## 문서 품질 기준
- 모든 Props에 설명과 예시 포함
- DO/DON'T 시각적 가이드 포함
- 코드 스니펫 복사-붙여넣기 가능
- 최신 코드와 문서 동기화 상태 유지

## DO
- Storybook, Docusaurus 등 전용 도구 활용
- 디자인 토큰 문서를 별도 섹션으로 관리
- 기여 가이드 포함 (새 컴포넌트 추가 절차)

## DON'T
- 문서 없이 컴포넌트 출시하지 않기
- 코드와 문서가 따로 관리되게 하지 않기
- 디자이너만 또는 개발자만 읽을 수 있게 쓰지 않기
