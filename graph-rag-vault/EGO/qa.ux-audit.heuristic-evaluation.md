# qa.ux-audit.heuristic-evaluation

## Content

휴리스틱 평가 (Nielsen의 10가지 원칙으로 사용성 문제를 체계적으로 진단한다):

### Nielsen's 10 Usability Heuristics
| # | 원칙 | 핵심 질문 |
|---|------|-----------|
| H1 | 시스템 상태 가시성 | 사용자가 현재 상태를 알 수 있는가? |
| H2 | 실세계 일치 | 시스템 용어가 사용자 언어인가? |
| H3 | 사용자 제어와 자유 | 실행 취소/되돌리기가 가능한가? |
| H4 | 일관성과 표준 | 같은 동작이 같은 결과를 내는가? |
| H5 | 에러 방지 | 실수를 사전에 방지하는 설계인가? |
| H6 | 기억보다 인식 | 옵션이 보이는가, 외워야 하는가? |
| H7 | 유연성과 효율 | 초보자와 전문가 모두 배려하는가? |
| H8 | 미니멀 디자인 | 불필요한 정보가 없는가? |
| H9 | 에러 인식/진단/복구 | 에러 메시지가 해결책을 안내하는가? |
| H10 | 도움말과 문서 | 필요 시 쉽게 찾을 수 있는가? |

### 심각도 등급 (Severity Rating)
| 등급 | 설명 | 예시 | 조치 |
|------|------|------|------|
| 0 | 사용성 문제 아님 | 개인적 취향 차이 | 무시 |
| 1 | 경미한 문제 | 버튼 라벨 불명확 | 여유 있을 때 |
| 2 | 주요 문제 | 필수 기능 찾기 어려움 | 높은 우선순위 |
| 3 | 대재앙 | 데이터 손실, 태스크 완료 불가 | 즉시 수정 |

### DO: 체계적 평가 프로세스
```
1. 준비 단계
   - 3-5명 독립 평가자 구성 (서로 다른 전문 분야)
   - 핵심 사용자 태스크 5-10개 정의
   - 각 평가자가 독립적으로 1-2시간 평가

2. 평가 수행
   - 각 원칙별로 위반 사항 기록
   - 위반마다 심각도 등급 부여 (0-3)
   - 스크린샷 + 구체적 위치 첨부

3. 통합 및 우선순위
   - 전체 발견 사항 통합 (중복 제거)
   - 심각도 × 빈도 = 우선순위 점수
   - P_score = severity × frequency (1-3)
   - P_score ≥ 6: 즉시 수정 (P1)
   - P_score 3-5: 다음 스프린트 (P2)
   - P_score 1-2: 백로그 (P3)
```

### 원칙별 진단 예시
```
■ H1 위반 — 시스템 상태 가시성
  위치: 파일 업로드
  문제: 업로드 진행률 표시 없음 → 사용자가 완료 여부 모름
  심각도: 2 (주요)
  개선: 프로그레스바 + 예상 시간 + 완료 알림

■ H5 위반 — 에러 방지
  위치: 회원 탈퇴 버튼
  문제: 확인 없이 즉시 탈퇴 실행 → 실수로 계정 삭제
  심각도: 3 (대재앙)
  개선: 2단계 확인 (모달 + 비밀번호 재입력)
```

DON'T:
```
// ❌ 단일 평가자 (개인 편향 → 발견 사항의 30-50%만 식별)
// 연구에 따르면 평가자 5명이면 문제의 75% 이상 발견

// ❌ 심각도 등급 없이 나열
"버튼 색상이 마음에 안 든다" ← 심각도? 빈도? 근거?

// ❌ 경미한 이슈를 대재앙보다 먼저 수정
아이콘 정렬(심각도 0) 수정에 1주 소요
→ 그 사이 데이터 손실(심각도 3) 방치
```

### Jakob's Law 적용
"사용자는 다른 사이트에서 익힌 관례를 기대한다"
→ 독창적 네비게이션보다 표준 패턴(GNB, 햄버거 메뉴) 사용
→ 결제 플로우는 업계 표준(배송지→결제→확인) 준수
