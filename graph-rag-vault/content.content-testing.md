# content.content-testing

> 콘텐츠 테스트는 A/B 테스트, 사용성 테스트, 가독성 테스트 등을 통해 콘텐츠 효과를 검증하는 과정이다. 데이터 기반 의사결정으로 직관에 의존하는 콘텐츠 결정을 줄인다.

# 콘텐츠 테스트 가이드

## 핵심 원칙
- 가설 기반 테스트 설계
- 한 번에 하나의 변수만 변경
- 통계적 유의성 확보 후 결론
- 테스트 결과를 지식 베이스에 축적

## 테스트 유형
### A/B 테스트
- 적용: 헤드라인, CTA, 이메일 제목
- 최소 샘플: 각 변형 1,000회 노출
- 기간: 최소 2주 (요일별 패턴 포함)

### 가독성 테스트
- 도구: Flesch-Kincaid, 한국어 가독성 지수
- 목표: 중학교 2학년 수준 이하
- 검증: 5초 테스트 (첫인상 + 핵심 메시지 파악)

### 사용성 테스트
- 도움말 문서: 태스크 완료율 측정
- 온보딩 텍스트: 이탈 지점 분석
- 에러 메시지: 자가 해결률 측정

## 테스트 프로세스
1. 가설 수립 ("CTA를 행동 중심으로 변경하면 클릭률이 15% 증가할 것")
2. 변형 제작 (원본 A vs 변형 B)
3. 트래픽 분배 (50:50)
4. 데이터 수집 (최소 2주)
5. 결과 분석 + 의사결정
6. 학습 기록

## DO
- 테스트 전 성공 기준 사전 정의
- 트래픽이 적으면 순차 테스트 적용
- 테스트 결과를 팀 전체에 공유

## DON'T
- 통계적 유의성 없이 승자 선언하지 않기
- 여러 변수를 동시에 변경하지 않기
- 테스트 중간에 조건 변경하지 않기
