# qa.test-gen.tdd

> TDD (Test-Driven Development) — Red-Green-Refactor 사이클

# TDD 가이드

## 핵심 원칙
- Red: 실패하는 테스트 먼저 작성
- Green: 최소 코드로 테스트 통과
- Refactor: 코드 개선 (테스트는 그대로 통과)

## DO
- 한 번에 하나의 테스트만 추가
- 테스트가 설계를 주도하게 허용
- 작은 단계로 점진적 구현

## DON'T
- 구현 먼저 작성 후 테스트 끼워맞추지 않기
- 리팩터링 단계를 건너뛰지 않기
- 너무 큰 단위로 테스트 작성하지 않기
