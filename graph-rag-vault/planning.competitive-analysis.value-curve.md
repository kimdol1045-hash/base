# planning.competitive-analysis.value-curve

> #83 Blue Ocean (Kim & Mauborgne, 2005)

Blue Ocean Strategy & Value Curve (경쟁 없는 새로운 시장 공간을 창출한다):

### 왜 Blue Ocean인가?
Red Ocean(기존 시장)에서 경쟁사와 같은 축으로 싸우면
가격 경쟁과 기능 군비 경쟁에 빠진다. Blue Ocean 전략은
경쟁의 축 자체를 바꿔서 "경쟁이 무의미한" 새로운 시장을 만든다.

### 가치 곡선(Value Curve) 작성법

**Step 1: 경쟁 요소 식별**
산업 내에서 경쟁이 일어나는 핵심 요소를 나열한다 (6-10개):
- 현재 경쟁사들이 투자하는 요소
- 고객이 비교할 때 사용하는 기준
- 예시 (프로젝트 관리 도구): 기능 수, 가격, 사용 편의성, 커스터마이징,
  통합(Integration) 수, 디자인, 협업 기능, 모바일 지원, 보안, 온보딩

**Step 2: 가치 곡선 그리기**
각 경쟁 요소에 대해 경쟁사와 우리의 수준을 1-5로 평가:

```
높음 5 │     *           *
     4 │   * │ *       * │
     3 │ * │ │ │ *   * │ │ *
     2 │ │ │ │ │ │ * │ │ │ │
낮음 1 │ │ │ │ │ │ │ │ │ │ │
       └─┴─┴─┴─┴─┴─┴─┴─┴─┴─
        가 사 커 통 디 협 모 보 온 기
        격 용 스 합 자 업 바 안 보 능
          편 터   인   일   딩 수
          의 마       지
          성 이       원
             징

── 경쟁사 A  -- 경쟁사 B  ** 우리 제품
```

**Step 3: 전략 캔버스 분석**
- 곡선이 겹치는 부분: Red Ocean (동일한 방식으로 경쟁 중)
- 곡선이 분리되는 부분: 차별화 기회
- 모든 경쟁사가 높은 부분: 과잉 투자 가능성 (줄일 기회)
- 모든 경쟁사가 낮은 부분: Underserved 기회

### ERRC 프레임워크 (제거-감소-증가-창조)

가치 곡선의 형태를 바꾸는 4가지 액션:

| 액션 | 정의 | 질문 |
|------|------|------|
| **Eliminate** (제거) | 업계 당연시하지만 실제로 불필요한 요소 | "이 기능이 정말 사용자에게 가치가 있는가?" |
| **Reduce** (감소) | 업계 표준 이하로 낮춰도 되는 요소 | "과잉 투자하고 있는 요소는?" |
| **Raise** (증가) | 업계 표준 이상으로 올려야 하는 요소 | "고객이 타협하고 있는 것은?" |
| **Create** (창조) | 업계에 없던 완전히 새로운 요소 | "제공된 적 없는 가치는?" |

### ERRC 템플릿
```
┌─────────────────────┬─────────────────────┐
│ Eliminate (제거)     │ Raise (증가)         │
│                     │                     │
│ - _____________     │ - _____________     │
│ - _____________     │ - _____________     │
│ - _____________     │ - _____________     │
├─────────────────────┼─────────────────────┤
│ Reduce (감소)       │ Create (창조)        │
│                     │                     │
│ - _____________     │ - _____________     │
│ - _____________     │ - _____________     │
│ - _____________     │ - _____________     │
└─────────────────────┴─────────────────────┘
```

### DO/DON'T 예시

**DO (밀키트 서비스 예시):**
- Eliminate: 오프라인 매장 운영 → 온라인 전용으로 비용 구조 혁신
- Reduce: 메뉴 다양성 (100개→15개) → 큐레이션으로 선택 피로 감소
- Raise: 조리 편의성 (전처리 수준을 업계 최고로)
- Create: "15분 요리 보장" → 시간 약속이라는 새로운 가치 축

**DON'T:**
- "모든 요소를 다 올린다" → 그것은 Blue Ocean이 아니라 비용 증가
- "경쟁사가 하는 것을 더 잘한다" → Red Ocean 접근
- Eliminate/Reduce 없이 Raise/Create만 나열
  → 반드시 줄이는 것이 있어야 집중이 가능

### 가치 곡선 비교표 템플릿
| 경쟁 요소 | 경쟁사A | 경쟁사B | 경쟁사C | 우리(AS-IS) | 우리(TO-BE) | ERRC |
|-----------|---------|---------|---------|-------------|-------------|------|
| 요소1 | 3 | 4 | 3 | - | 1 | Eliminate |
| 요소2 | 4 | 4 | 5 | - | 2 | Reduce |
| 요소3 | 2 | 3 | 2 | - | 5 | Raise |
| 요소4 | 0 | 0 | 0 | - | 4 | Create |

### 주의사항
- 경쟁 요소는 고객 관점에서 선정 (기술 내부 요소가 아닌 사용자 가치)
- "우리만의 새로운 축"이 최소 1개는 있어야 Blue Ocean
- ERRC 4가지 중 최소 3가지는 채워야 전략적 의미가 있음
- 가치 곡선의 점수는 주관적이므로 "[팀 합의 기반]" 명시
