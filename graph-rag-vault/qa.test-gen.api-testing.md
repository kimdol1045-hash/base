# qa.test-gen.api-testing

> API 테스트는 애플리케이션 인터페이스의 기능, 성능, 보안을 검증하는 테스트로, UI 없이 비즈니스 로직을 직접 테스트한다. 테스트 피라미드에서 통합 테스트 레벨에 해당하며, E2E보다 빠르고 안정적이다.

# API 테스트 가이드

## 핵심 원칙
- Happy Path + Edge Case + Error Case 모두 커버
- 상태 코드, 응답 바디, 헤더 모두 검증
- 테스트 데이터 격리 (테스트 간 독립성)
- 인증/인가 시나리오 필수 포함

## 테스트 범위
| 영역 | 검증 항목 |
|------|----------|
| 기능 | 올바른 응답 데이터, 상태 코드 |
| 검증 | 필수 필드 누락, 잘못된 타입 |
| 인증 | 토큰 없음, 만료, 권한 부족 |
| 성능 | 응답 시간, 동시 요청 |
| 에러 | 404, 500, 타임아웃 처리 |

## 테스트 구조
```
describe("POST /api/users")
  it("201: 유효한 데이터로 사용자 생성")
  it("400: 이메일 누락 시 검증 에러")
  it("400: 잘못된 이메일 형식")
  it("409: 중복 이메일")
  it("401: 인증 토큰 없음")
  it("403: 권한 부족")
```

## 테스트 데이터 관리
- Factory 패턴으로 테스트 데이터 생성
- 테스트 후 정리(Cleanup) 자동화
- 시드 데이터와 동적 데이터 분리

## DO
- 모든 엔드포인트의 에러 응답 형식 일관성 검증
- Rate Limit, 페이지네이션 경계값 테스트
- CI에서 매 PR마다 자동 실행

## DON'T
- 프로덕션 데이터로 테스트하지 않기
- Happy Path만 테스트하지 않기
- 테스트 간 상태 의존성 만들지 않기
