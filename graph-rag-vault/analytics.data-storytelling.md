# analytics.data-storytelling

> 데이터 스토리텔링은 데이터 분석 결과를 서사 구조와 시각화를 결합하여 설득력 있게 전달하는 기술이다. Cole Nussbaumer Knaflic의 프레임워크에 따르면 맥락(Context) + 데이터(Data) + 서사(Narrative)의 3요소가 핵심이다.

# 데이터 스토리텔링 가이드

## 핵심 원칙
- 데이터는 근거, 스토리는 전달 수단
- "그래서 어쩌라고?(So What?)"에 답하는 인사이트
- 청중의 지식 수준과 관심사에 맞춤
- 시각화는 스토리를 강화하는 도구

## 스토리 구조
1. **배경 설정**: 왜 이 분석을 했는가?
2. **현상 제시**: 데이터가 보여주는 것
3. **인사이트**: 데이터 뒤에 숨은 의미
4. **권고 사항**: 이 데이터를 바탕으로 무엇을 해야 하는가?
5. **다음 단계**: 추가 분석 또는 액션 플랜

## 시각화 원칙
- 하나의 차트에 하나의 메시지
- 주목해야 할 부분을 시각적으로 강조
- 불필요한 시각적 요소 제거
- 차트 제목에 인사이트를 포함

## 발표 기법
- 차트를 보여주기 전 맥락 설명
- 핵심 수치를 말로 한 번 더 강조
- 청중의 질문을 예상하여 부록 준비

## DO
- 핵심 인사이트를 3개 이내로 압축
- 비교 기준 (전년 동기, 업계 평균) 제시
- 액션 아이템을 명확하게 제안

## DON'T
- 데이터 덤프 (차트를 나열만 하기) 하지 않기
- 분석 과정을 발표에 그대로 옮기지 않기
- 인사이트 없이 "현황 보고"만 하지 않기
