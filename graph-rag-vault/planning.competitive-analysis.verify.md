# planning.competitive-analysis.verify

경쟁 분석 자기 검증 체크리스트 (분석 완성 후 반드시 실행):

### 왜 경쟁 분석 검증이 필요한가?
경쟁 분석은 확증편향에 가장 취약한 작업이다. 무의식적으로
우리 제품에 유리한 정보만 수집하거나, 경쟁사를 과소평가하는
경향이 있다. 체계적 검증으로 객관성을 확보해야 한다.

### 데이터 신뢰성 검증
- [ ] 모든 수치(MAU, 다운로드 수, 매출)에 출처가 있는가?
- [ ] 출처 없는 수치에 [추정치] 또는 [검증 필요] 태그가 있는가?
- [ ] 인용한 기업/서비스가 실제로 존재하는가? (AI 환각 체크)
- [ ] 앱스토어 리뷰, 공식 웹사이트 등 공개 데이터에 기반하는가?
- [ ] 1년 이상 된 데이터에 시점이 명시되어 있는가?

### 균형성 검증
- [ ] 경쟁사의 장점과 단점이 균형 있게 제시되는가?
  → 장점만 3개, 단점 0개인 경쟁사가 있다면 재분석
- [ ] 우리 제품의 약점도 솔직하게 인정되어 있는가?
- [ ] "우리가 무조건 이긴다"는 톤이 없는가?
- [ ] 경쟁사가 우리보다 나은 점을 최소 1개 이상 식별했는가?

### 분석 범위 검증
- [ ] 직접 경쟁사가 3-5개 범위 내인가? (3개 미만이면 시장 조사 부족)
- [ ] 간접 경쟁사/대체재가 포함되어 있는가? (엑셀, 수기, 무소비 등)
- [ ] 포지셔닝 맵의 축이 고객 관점에서 의미 있는 기준인가?
  → "기능 많음 vs 기능 적음"은 고객 가치가 아닐 수 있다
  → "사용 편의성 vs 기능 깊이" 같은 트레이드오프 축이 더 유용
- [ ] 포지셔닝 맵에 빈 사분면(기회 영역)이 식별되었는가?

### Porter 5 Forces 검증 (해당 시)
- [ ] 5개 Force 모두 분석되어 있는가?
- [ ] 각 Force에 높/중/낮 수준과 구체적 근거가 있는가?
- [ ] 종합 판단이 개별 Force 분석과 일관성이 있는가?
- [ ] 대체재 분석에 비전통적 경쟁(엑셀, 카톡 등)이 포함되어 있는가?

### SWOT 검증 (해당 시)
- [ ] 내부 요인(S/W)과 외부 요인(O/T)이 구분되어 있는가?
- [ ] 각 사분면에 3-5개 항목이 있는가?
- [ ] Strength가 실제 검증 가능한 역량인가? (희망사항이 아닌가?)
- [ ] Threat에 현실적이고 구체적인 위협이 포함되어 있는가?

### 전략적 연결 검증
- [ ] 차별화 기회가 실제 고객 불만(리뷰)에서 도출되었는가?
- [ ] 진입 전략이 분석 결과와 논리적으로 연결되는가?
- [ ] 경쟁사 예상 대응이 고려되어 있는가?

### 검증 실패 시 조치
- 데이터 신뢰성 미통과: 해당 수치 제거하거나 [추정치] 태그 추가
- 균형성 미통과: 경쟁사 장점과 우리 약점을 추가 분석
- 분석 범위 미통과: 누락된 경쟁사/대체재 추가
- 전략 연결 미통과: 분석 → 전략 논리 재구성
