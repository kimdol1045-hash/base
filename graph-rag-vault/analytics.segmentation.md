# analytics.segmentation

> 사용자 세그먼테이션은 공통 특성을 가진 사용자를 그룹으로 분류하여 각 그룹에 맞는 전략을 수립하는 기법이다. RFM(최근성, 빈도, 금액), 행동 기반, 가치 기반 세분화가 대표적이다.

# 사용자 세그먼테이션 가이드

## 핵심 원칙
- 세그먼트는 실행 가능(Actionable)해야 한다
- 행동 기반 세분화가 인구통계보다 예측력 높음
- 세그먼트별로 다른 KPI와 전략 적용
- 세그먼트는 정적이 아니라 동적 — 주기적 갱신

## 세그먼테이션 방법
### RFM 분석
- Recency: 최근 활동일
- Frequency: 활동 빈도
- Monetary: 지출 금액
- 각 지표를 1-5 점수화 → 125개 조합

### 행동 기반
- 사용 패턴: 파워유저, 일반, 저활성
- 기능 사용: 핵심 기능 사용 여부
- 여정 단계: 온보딩, 활성화, 숙련

### 가치 기반
- 고LTV / 저LTV 구분
- 성장 잠재력 (현재 가치 vs 미래 가치)

## DO
- 세그먼트별 대시보드 구성
- 세그먼트 이동(Migration) 추적
- 마케팅/제품 전략에 세그먼트 반영

## DON'T
- 10개 이상 세그먼트로 과도하게 세분화하지 않기
- 인구통계만으로 세그먼트하지 않기
- 세그먼트를 한 번 만들고 갱신하지 않기
