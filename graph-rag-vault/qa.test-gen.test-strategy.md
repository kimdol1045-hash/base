# qa.test-gen.test-strategy

> 테스트 전략 — 리스크 기반 우선순위와 커버리지 최적화

# 테스트 전략 수립

## 핵심 원칙
- 리스크 기반 우선순위: 비즈니스 임팩트 × 장애 확률
- 테스트 피라미드/트로피 구조에 맞는 레벨별 배분
- 자동화 가능 영역과 수동 탐색 영역 분리

## DO
- 핵심 사용자 여정 기반 E2E 시나리오 정의
- 회귀 리스크 높은 영역 우선 자동화
- 테스트 매트릭스로 커버리지 시각화

## DON'T
- 100% 커버리지 맹목 추구하지 않기
- UI 테스트에만 의존하지 않기
- 테스트 유지보수 비용 무시하지 않기
