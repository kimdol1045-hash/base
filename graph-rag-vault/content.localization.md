# content.localization

> 로컬라이제이션은 단순 번역을 넘어 문화적 맥락, 법적 요건, UX 관습을 현지화하는 과정이다. CSA Research에 따르면 현지화된 콘텐츠는 구매 전환율을 72% 높인다.

# 로컬라이제이션 가이드

## 핵심 원칙
- 번역(Translation)과 현지화(Localization) 구분
- 문화적 맥락 고려 (색상, 이미지, 관용표현)
- 기술적 국제화(i18n) 인프라 선행 구축
- 용어집(Glossary) + 스타일 가이드 우선 확보

## 프로세스
1. **국제화**: 하드코딩 문자열 추출, 날짜/통화 포맷 분리
2. **용어집 구축**: 핵심 용어 50-100개 통일
3. **번역**: 전문 번역가 + CAT 도구 활용
4. **현지화 검수**: 네이티브 리뷰어 QA
5. **문화적 적합성**: 이미지, 색상, 레이아웃 조정
6. **기능 테스트**: 텍스트 확장/축소에 따른 UI 깨짐 확인

## DO
- 텍스트 확장율 고려 (영→한: +15%, 영→독: +30%)
- 날짜/시간/통화 포맷을 locale 기반으로 자동 처리
- 복수형 규칙이 다른 언어 고려 (아랍어 6개 복수형)
- 스크린샷 포함 컨텍스트를 번역가에게 제공

## DON'T
- 문자열 연결(concatenation)로 문장 조립하지 않기
- 문화적 금기 무시하지 않기 (색상, 제스처 등)
- 기계 번역 결과를 검수 없이 배포하지 않기
- RTL(오른쪽→왼쪽) 언어 레이아웃 미지원
