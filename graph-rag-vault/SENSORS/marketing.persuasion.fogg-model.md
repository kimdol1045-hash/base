# marketing.persuasion.fogg-model

> **Region**: 📡 [[SENSORS]]  
> **Domain**: `marketing`  
> **Type**: `rule`  
> **Theory**: #67 포그 행동 모델 (Fogg, 2009)  
> **Tokens**: 500

## Content

포그 행동 모델 — B = M × A × P (행동 = 동기 × 능력 × 촉발):

### 핵심 원리
세 요소가 동시에 충분해야 행동이 일어난다.
하나라도 부족하면 행동은 일어나지 않는다.

### 1. Motivation (동기) — 왜 해야 하는가?
동기의 3가지 축:
- 쾌락/고통: "이 도구로 야근이 사라집니다" / "수동 작업의 고통에서 벗어나세요"
- 희망/두려움: "매출 3배 성장" / "경쟁사에 뒤처질 수 있습니다"
- 소속/거부: "12,000팀이 함께합니다" / "아직도 엑셀로 하시나요?"

**동기 높이는 카피 패턴**:
- 고통 환기: "매달 리포트 작성에 3일을 쓰고 계시죠?"
- 이점 제시: "자동화하면 3일이 30분으로 줄어듭니다"
- 사회적 압력: "같은 업종의 78%가 이미 자동화했습니다"
- 손실 회피: "매월 120시간의 인건비를 낭비하고 있습니다"

### 2. Ability (능력) — 얼마나 쉬운가?
단순화의 6요소 (하나라도 어려우면 행동 안 함):
| 요소 | 높은 능력 (DO) | 낮은 능력 (DON'T) |
|------|---------------|------------------|
| 시간 | "30초면 완료" | 가입 폼 15개 항목 |
| 비용 | "무료로 시작" | 최소 결제 ₩99,000 |
| 체력 | 버튼 1번 클릭 | 5페이지 스크롤 후 CTA |
| 인지 | "이메일만 입력" | 복잡한 설정 필요 |
| 사회적 일탈 | "12,000명이 사용 중" | "새로운 시도" 강조 |
| 비일상 | 기존 습관에 통합 | 완전히 새로운 워크플로우 |

**능력 높이는 UX 패턴**:
- 소셜 로그인: "구글로 3초 만에 시작"
- 단계 분리: 한 화면에 1가지 행동만 요구
- 기본값 설정: 최적 옵션 미리 선택
- 프리필: 이전 입력 자동 완성
- 프로그레스 바: "3단계 중 2단계"

### 3. Prompt (촉발) — 지금 해야 하는 이유?
촉발의 3유형:
| 유형 | 상황 | 예시 |
|------|------|------|
| Spark | 동기 낮음 + 능력 높음 | 영감 주는 카피, 후기, 사례 |
| Facilitator | 동기 높음 + 능력 낮음 | "원클릭으로 시작", 단순화 |
| Signal | 동기 높음 + 능력 높음 | 단순 리마인더, 알림 |

**촉발 타이밍**:
- 가입 직후: 온보딩 메일 (5분 이내)
- 비활성 3일 후: "다시 돌아오세요" 이메일
- 기능 사용 직후: 인앱 업셀 ("이 기능 더 쓰려면 Pro")
- 목표 달성 시: "축하합니다!" + 다음 행동 제안

### CTA 설계 공식
```
[동기 카피] + [능력 보장] + [촉발 버튼]

"매출 데이터를 한눈에 파악하세요"     ← 동기 (이점)
"설정 없이 바로 연동됩니다"           ← 능력 (쉬움)
[무료로 시작하기]                     ← 촉발 (행동 유도)
```

### 전환율이 낮을 때 진단 체크리스트
- [ ] 동기 부족? → 카피에서 이점/고통이 명확한가?
- [ ] 능력 부족? → 행동까지 클릭 수가 3회 이내인가?
- [ ] 촉발 부재? → CTA가 눈에 잘 보이는가? 타이밍이 맞는가?
