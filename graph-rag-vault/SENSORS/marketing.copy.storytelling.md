## Content

브랜드 스토리텔링 (고객이 주인공, 브랜드는 가이드):

### StoryBrand 프레임워크 (Miller, 2017)
1. **주인공**: 고객 (브랜드가 아님!)
2. **문제**: 외적 문제 + 내적 문제 + 철학적 문제
3. **가이드**: 브랜드 (공감 + 권위)
4. **계획**: 3단계 간단한 실행 계획
5. **행동 촉구**: 명확한 CTA
6. **실패 회피**: 안 하면 어떻게 되는지
7. **성공**: 하면 어떻게 달라지는지

### 예시 (프로젝트 관리 SaaS)
- 주인공: "스타트업 CTO인 당신은"
- 외적 문제: "프로젝트 일정이 항상 밀립니다"
- 내적 문제: "팀원들 앞에서 무능하게 보일까 불안합니다"
- 가이드: "저희도 5년간 같은 고민을 했습니다 (공감). 200개 팀의 프로젝트를 분석했습니다 (권위)."
- 계획: "1) 가입 → 2) 프로젝트 연동 → 3) 자동 리포트 확인"
- 행동: "14일 무료 체험 시작하기"
- 실패: "계속 수동 관리하면 번아웃이 옵니다"
- 성공: "매주 5시간을 절약하고, 팀의 신뢰를 얻으세요"

### 규칙
- 브랜드 자랑 ❌ → 고객의 변화 ✅
- 한 문장으로: "[고객]이 [문제]를 [해결]하여 [결과]를 얻는 이야기"
