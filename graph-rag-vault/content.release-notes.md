# content.release-notes

> 릴리즈 노트는 제품 변경 사항을 사용자에게 전달하는 커뮤니케이션 도구이다. 단순 변경 로그를 넘어 사용자 가치 중심으로 작성하면 기능 도입률(Feature Adoption)이 크게 향상된다.

# 릴리즈 노트 작성 가이드

## 핵심 원칙
- 기술 변경이 아닌 사용자 가치 중심 서술
- 카테고리별 분류: 신기능/개선/수정/제거
- 시각적 변경은 스크린샷 또는 GIF 포함
- 일관된 발행 주기와 포맷 유지

## 릴리즈 노트 구조
1. **버전 + 날짜**: v2.4.0 — 2024년 3월 15일
2. **하이라이트**: 가장 중요한 변경 1-2개 강조
3. **신기능** (New): 새로 추가된 기능
4. **개선** (Improved): 기존 기능 개선
5. **수정** (Fixed): 버그 수정
6. **제거** (Removed): 더 이상 지원하지 않는 기능
7. **마이그레이션 가이드**: 필요 시 업데이트 방법

## 작성 포맷
```
### 신기능
- **다크 모드 지원**: 설정 > 테마에서 다크 모드를 선택할 수 있습니다.
  눈의 피로를 줄이고 배터리 사용량을 절약합니다.
```

## DO
- "~할 수 있습니다" 형태로 사용자 행동 중심 서술
- Breaking Change는 상단에 경고로 표시
- 관련 도움말 문서 링크 첨부

## DON'T
- 커밋 메시지를 그대로 나열하지 않기
- 내부 이슈 번호만 기재하지 않기
- 사용자에게 영향 없는 내부 리팩토링 포함하지 않기
